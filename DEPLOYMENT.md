# Інструкція з розгортання на GitHub Pages

## Крок 1: Створіть GitHub репозиторій

1. Перейдіть на [GitHub](https://github.com) і увійдіть у свій акаунт
2. Натисніть "New repository" (зелена кнопка)
3. Назвіть репозиторій (наприклад, `lifematika-landing`)
4. Зробіть його **Public** (обов'язково для безкоштовного GitHub Pages)
5. **НЕ** додавайте README, .gitignore, або ліцензію (вони вже є)
6. Натисніть "Create repository"

## Крок 2: Підготуйте локальний репозиторій

Відкрийте термінал у папці проєкту і виконайте команди:

```bash
# Ініціалізуйте git репозиторій
git init

# Додайте всі файли
git add .

# Зробіть перший коміт
git commit -m "Initial commit: Lifematika landing page"

# Додайте віддалений репозиторій (замініть YOUR_USERNAME та YOUR_REPO на свої)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# Перейменуйте гілку на main (якщо потрібно)
git branch -M main

# Завантажте код на GitHub
git push -u origin main
```

## Крок 3: Налаштуйте GitHub Pages

1. Перейдіть у свій репозиторій на GitHub
2. Натисніть на **Settings** (Налаштування)
3. У лівому меню знайдіть **Pages**
4. У розділі **Source** виберіть:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
5. Натисніть **Save**
6. Зачекайте 1-2 хвилини

## Крок 4: Перевірте розгортання

Після збереження GitHub покаже URL вашого сайту:
```
https://YOUR_USERNAME.github.io/YOUR_REPO/
```

Ваш сайт буде доступний за цією адресою!

## Налаштування власного домену (опціонально)

Якщо у вас є власний домен:

1. У налаштуваннях GitHub Pages введіть ваш домен у поле **Custom domain**
2. У налаштуваннях вашого DNS провайдера додайте CNAME запис:
   - **Type**: CNAME
   - **Name**: www (або @)
   - **Value**: YOUR_USERNAME.github.io

3. Для кореневого домену додайте A записи:
   ```
   185.199.108.153
   185.199.109.153
   185.199.110.153
   185.199.111.153
   ```

## Оновлення сайту

Щоб оновити сайт після змін:

```bash
git add .
git commit -m "Опис ваших змін"
git push
```

Зміни з'являться на сайті протягом 1-2 хвилин.

## Підказки

- Файл `.nojekyll` вже створено - він потрібен для правильної роботи GitHub Pages
- Всі зміни автоматично публікуються після `git push`
- Можна налаштувати HTTPS у налаштуваннях GitHub Pages (рекомендовано)
